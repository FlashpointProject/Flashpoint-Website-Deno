## Flashpoint Launcher

I'm breaking this up into 3 sections
- **Performance and Maintainability** - Changes to make the launcher faster and easier to maintain
- **Usability** - Changes shipping with the launcher that most users should notice and benefit from
- **Extensions** - A fairly comprehensive look at what's changing with extensions and why it'll benefit most users

## 🛠️ Performance and Maintainability

This has been a huge and frankly time consuming part of development for the past 6+ months. Whilst this matters much more if you're a developer, this all has a knock on effect to making everything else easier to work on.
- Stripped out almost all manual memoization so we can use the new React Compiler for easier to read code and performance improvements
- Added support for running in a browser during development for hot reloading of components and more modern dev tools without being restricted to an ancient version of Electron
- Added context isolation and removed node integration from the renderer, which has been an ongoing job for years
- Restructured most components to use the application store where possible instead of passing down props and avoid hard to diagnose re-renders
- Rewrote most of the remaining class components into more friendly functional components
- Shrunk the launcher by around 20% by bundling the backend, main and preload sections ahead of packaging
